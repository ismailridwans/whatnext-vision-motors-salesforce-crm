Project Breakdown

Explore the solution from three angles.

Customer Ordering

Retail orders confirm immediately only when stock can be reserved. Bulk orders may stay pending until batch recovery finds enough inventory.

Dealer Assignment

Orders route using coordinates first, then postal code, then state. That keeps assignment practical even when only partial customer data is available.

After-Sales Support

Service requests and test drives live in the same app, so the project tells a full customer journey story instead of a narrow order-only demo.

Apex Trigger Framework

Trigger handler classes separate routing, inventory allocation, and notification logic so the automation stays maintainable.

Batch + Scheduled Apex

The inventory batch refreshes stock availability, resolves pending bulk orders, and sends low-stock alerts. Another scheduler handles test-drive reminders.

Deployment Assets

Layouts, a permission set, demo data scripts, and runbooks make the repo useful in a real org setup, not just in code review.
